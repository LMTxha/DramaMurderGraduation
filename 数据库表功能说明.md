# DramaMurderGraduationDb 数据库表功能说明

本文档用于说明 `DramaMurderGraduationDb` 数据库中主要数据表与项目功能模块之间的对应关系。该数据库是剧本杀门店运营与玩家服务系统的核心数据来源，支撑用户登录、剧本展示、预约下单、钱包充值、DM 游戏流程、好友社交、后台审核和数据分析等功能。

## 1. 数据库基本信息

| 项目 | 内容 |
|---|---|
| 数据库名称 | `DramaMurderGraduationDb` |
| 数据库类型 | SQL Server LocalDB |
| 项目连接配置 | `DramaMurderGraduation.Web/Web.config` |
| 默认连接服务器 | `(localdb)\MSSQLLocalDB` |
| 数据文件 | `DramaMurderGraduationDb.mdf` |
| 日志文件 | `DramaMurderGraduationDb_log.ldf` |

项目运行时连接字符串中的数据库名为：

```text
Initial Catalog=DramaMurderGraduationDb
```

因此，本项目正常运行时只需要保留并连接 `DramaMurderGraduationDb` 这个数据库。

## 2. 用户与账号模块

| 数据表 | 对应功能 |
|---|---|
| `Users` | 用户账号、登录信息、角色权限、审核状态、余额等核心用户数据 |
| `PasswordResetTickets` | 忘记密码、密码重置验证码或重置凭证 |
| `UserLoginSecurityLogs` | 用户登录安全记录，例如登录时间、登录状态等 |
| `UserProfileChangeLogs` | 用户资料修改记录 |
| `UserDesktopSettings` | 用户桌面端或聊天工作台相关设置 |
| `UserNotificationReads` | 用户通知已读记录 |
| `UserQuickNotes` | 用户快捷备注、个人便签 |
| `UserBlocks` | 用户拉黑或屏蔽关系 |

## 3. 玩家档案与成长模块

| 数据表 | 对应功能 |
|---|---|
| `PlayerProfiles` | 玩家个人主页、昵称、头像、称号、偏好类型等 |
| `PlayerAbilities` | 玩家能力值，例如推理力、观察力、协作力等 |
| `PlayerBattleRecords` | 玩家游戏战绩、完成记录、胜率相关数据 |
| `Achievements` | 成就勋章墙 |

## 13. 总结

从功能上看，`DramaMurderGraduationDb` 数据库可以分为以下几类：

| 模块 | 代表数据表 |
|---|---|
| 用户账号 | `Users`、`PasswordResetTickets`、`UserLoginSecurityLogs` |
| 玩家档案 | `PlayerProfiles`、`PlayerAbilities`、`Achievements` |
| 剧本内容 | `Scripts`、`Genres`、`ScriptCharacters`、`ScriptClues` |
| 场次预约 | `Rooms`、`Sessions`、`Reservations` |
| 钱包订单 | `WalletTransactions`、`RechargeRequests`、`AfterSaleRequests` |
| 游戏流程 | `GameStages`、`SessionGameStates`、`SessionVotes` |
| 社交聊天 | `FriendMessages`、`GroupMessages`、`RoomMessages` |
| 观战直播 | `SpectatorRooms`、`SpectatorMessages`、`LiveSessions` |
| 后台分析 | `AnalyticsSnapshots`、`BusinessActionLogs`、`EconomyInsights` |
| 展示页面 | `ShowcasePages`、`ShowcaseSections`、`ShowcaseEntries` |

这些表共同构成项目的数据支撑。正常情况下不要删除 `DramaMurderGraduationDb` 中的任何业务表，否则对应页面可能无法正常加载或执行。
